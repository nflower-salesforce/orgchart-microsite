# Org Chart Microsite

A password-protected, interactive, **editable** stakeholder org chart you can
stand up on Heroku in a couple of minutes. Drag to reposition, add/edit people
right in the browser, and switch between horizontal, vertical, concentric, and
network layouts. Data persists in Postgres.

---

## 1. Deploy it (no command line needed)

> Just click the button — it deploys *your own* copy onto *your* Heroku account
> (the database, password, and data are all yours). You only need to **fork this
> repo** first if you want to change the code; to customize the roster, you can
> do it in the browser or via `seed-data.js` after deploying.

[![Deploy](https://www.herokucdn.com/deploy/button.svg)](https://heroku.com/deploy?template=https://github.com/nflower-salesforce/orgchart-microsite)

Clicking the button opens a Heroku form. Fill in:

| Field | What to enter |
| --- | --- |
| **App name** | Anything unique, e.g. `acme-org-chart` |
| **SITE_PASSWORD** | The password you'll share with your team (change `change-me`) |
| **SESSION_SECRET** | Leave as-is — it auto-generates |
| **SITE_TITLE** / **SITE_SUBTITLE** | Optional header text |

Heroku provisions a small Postgres database, builds the app, and seeds it with a
3-person example. Click **View** → log in with your password → you're live.

That's it. Everything from here is optional.

---

## 2. Make it yours

- **Add people in the browser.** Hover a card → the **＋ (add report)** button;
  the **Add person** button (top right) for a new top-level person. Drag cards to
  reposition; click a card to open/edit. No code required.
- **Start from your real org.** The seed (`seed-data.js`) only populates an
  *empty* database on first boot. The easy path: open this repo in
  [Claude Code](https://www.anthropic.com/claude-code), hand it a screenshot or
  spreadsheet of your org, and ask it to rewrite `seed-data.js`. Then redeploy
  (or reset — see below).
- **Re-seed after editing `seed-data.js`.** Seeding only runs on an *empty* DB.
  To reset: in the Heroku dashboard, **Resources → Heroku Postgres → Settings →
  Reset Database**, then restart the dyno.
- **Branding.** Set config vars in **Settings → Config Vars**:
  `SITE_TITLE`, `SITE_SUBTITLE`, and `SITE_LOCKUP` (a URL to a logo image shown
  in the header).

---

## 3. Run it locally (optional)

```bash
npm install
SITE_PASSWORD=secret npm start      # → http://localhost:3000
npm test                            # run the test suite
```

With no `DATABASE_URL` set, it uses an in-memory store (data resets on restart),
so you can experiment freely without a database.

---

## How it works

- **`app.js`** — Express server: a password gate (one shared password), the
  org-chart page, and a small JSON API (`/api/stakeholders`).
- **`data.js`** — storage layer. Uses **Heroku Postgres** when `DATABASE_URL` is
  set, otherwise an in-memory store. Auto-creates the schema and seeds it from
  `seed-data.js` the first time it finds the database empty.
- **`org-chart.html`** — the entire UI (SVG, no build step, no external JS libs).
- **`seed-data.js`** — the example roster; only used to seed an empty database.

Config (Heroku Config Vars / local env):

| Var | Purpose |
| --- | --- |
| `SITE_PASSWORD` | **Required.** Password to view the site. |
| `SESSION_SECRET` | Signs login sessions (auto-generated by the button). |
| `DATABASE_URL` | Set automatically by the Heroku Postgres add-on. |
| `SITE_TITLE` / `SITE_SUBTITLE` / `SITE_LOCKUP` | Header branding (optional). |
